02.项目运行环境及项目经理角色

Pupper2024-07-102024-09-20

1. 项目运行环境

1.1 事业环境因素

事业环境因素 : 通常源于外部环境, 是指对项目、项目管理团队产生影响、限制或指令作用的各种条件。

  • 组织内部的事业环境因素 : 包含基础设施、资源可用性、组织文化、结构和治理、信息技术、员工能力等
  • 组织外部的事业环境因素 : 包含市场条件、法律法规、行业标准、竞争情况、社会和文化影响等

1.2 组织过程资产

组织过程资产 : 是组织所特有并使用的计划、过程、政策和知识库, 会影响对具体项目的管理。

  • 过程、政策、程序 : 组织用于执行项目工作的流程与程序, 包括组织的标准、规范、过程、程序和政策等
  • 知识库 : 包括已经积累的经验教训、历史信息、过程资料、模板、指导文件等
事业环境因素组织过程资产
1. 必须考虑内外部环境、条件
2. 项目 极少 对其产生影响
3. 直接约束项目(积极或消极)		1. 供项目利用的、支持作用的因素
2. 项目的实际会引起其更新
3. 组织内部所有, 需要主动利用

1.3 组织系统

  • 职能型组织: 基于职能的决策授权组织, 即为弱矩阵型组织
  • 项目型组织: 基于项目的决策授权组织, 即为强矩阵型组织
  • 矩阵型组织: 介于职能型组织和项目型组织之间的组织形式
组织结构类型优点缺点本质
职能型组织职责分工明确无全职项目员工, 项目工作容易被忽视临时搭伙
项目型组织项目经理权利充分,对资源有控制权资源重复配置,资源使用效率低全职干活, 完工解散
矩阵型组织兼顾项目与职能工作,提高资源的使用效率沟通与管理工作复杂两头汇报

1.3.1 项目管理办公室(PMO)

项目管理办公室(PMO) : 是一个组织实体, 负责项目管理的相关职能, 为项目经理提供支持和指导。

  • 支持型 PMO : 担当顾问和资源库的角色,提供模板、最佳实践、培训、以及来自其他项目的经验教训等
  • 控制型 PMO : 除了提供支持外, 还会对项目进行监控, 要求项目服从.
  • 指令型 PMO : 直接管理和控制项目, 项目经理由 PMO 指定并向其汇报.

为了保证项目符合组织的业务目标, PMO 可能有权在每个项目的生命周期中充当 重要相关方和决策者 的角色。
PMO 的主要职能是通过各种方式向项目经理提供支持, 包括但不限于指导、监督、提供资源、跨项目沟通等等, 以确保项目成功。

2. 项目经理角色

项目经理 : 是负责项目实现目标的人, 他/她负责项目的计划、执行和收尾, 以及项目团队的管理。

搞项目 = 搞人脉 + 搞关系 + 搞资源 + 搞工作

管理一个项目包含:

  • 识别制约和目标, 并带领团队进行分解、计划、实施
  • 平衡质量、范围、时间、成本四方面互不相让的要求
  • 整合各领域知识、技术与资源, 交付特定成果
  • 引导并满足项目相关方不同的需求与期望
  • 影响相关组织和内外部环境,引导组织变革和优化

2.1 项目经理的影响力

  • 对 项目 的影响: 领导项目团队, 管理相关方期望, 利用可用资源, 充当发起人、成员与相关方之间的平衡者
  • 对 组织 的影响: 与其他部门沟通, 展现项目管理的价值, 提高组织对项目管理的接受度, 提高组织内现有 PMO 的效率.

2.3 :bell:项目经理的能力

  • 技术项目管理 : 与项目、项目集和项目组合管理特定领域相关的知识、技能和行为, 即角色履行的技术方面
    • 重点关注项目的关键要素与指标, 如 项目成功的关键因素,特定的财务报告等
    • 针对项目裁剪传统或敏捷工具、技术和方法
    • 管理项目各个要素: 进度、成本、质量、资源、风险、采购、沟通、干系人等
  • 领导力 : 指导、激励和带领团队所需的知识、技能和行为, 可帮助组织达成业务目标
  • 战略和商务管理 : 关于行业和组织的知识和专业技能, 有助于提高绩效并取得更好的业务成果

2.4 管理与领导力的比较

  • 管理 : 运用一系列一直的预期行为 指示 他人, 以实现共同目标
  • 领导力 : 通过讨论或辩论与他人合作, 带领 他人实现共同目标

领导力风格:

  • 放任型领导 : 允许团队自主决策和设定目标
  • 交易型领导 : 关注目标、反馈和成就以确定奖励
  • 服务型领导 : 做出服务承诺, 关注他人成长、发展, 关注人际关系、团队合作
  • 变革型领导 : 通过理想化特质和行为、鼓舞、激励、促进创新和变革
  • 魅力型领导 : 通过个人魅力、信任、尊重、情感吸引他人
  • 交互型颖导 : 结合了交易型、变革型和魅力型领导的特点

2.5 敏捷环境下的项目经理

敏捷项目经理 : 与传统项目经理相比, 敏捷项目经理更注重团队的自组织、自管理和自领导, 以及与相关方的协作和沟通

预测型 PM 和敏捷型 PM 的比较:

~预测 PM敏捷 PM
定位最终责任人, 项目中心提供服务、促进合作者、非项目中心
~管理过程,协调团队服务团队
职责获取资源,组建团队引导组建团队,引导团队成员达成共识
~规划项目教成员规范做法
~管理过程监督成员按照规范做,洞察风险
~管理团队,协调相关方帮助团队解决问题,排除障碍
作用聚焦目标促进合作
~管理协调排除障碍
~建设管理团队引导提升团队
